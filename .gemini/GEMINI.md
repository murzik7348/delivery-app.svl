## API

- Авторизація: Bearer token через Axios.
- **Статуси доставки**: 0=created, 1=accepted, 2=preparing, 3=delivering, 4=delivered, 5=canceled.
- **Endpoint'и**:
  - `GET /admin/delivery` — список замовлень
  - `DELETE /admin/delivery/{id}` — видалення замовлення
  - `PUT /restaurant/deliveries/{id}/confirm` — підтвердження (тіло `{}`)
  - `PUT /restaurant/deliveries/{id}/cancel` — скасування
  - Кур’єр: `/courier/deliveries/{id}/accepted`, `.../pickedup`, `.../confirmations`

## Важливо

- Для `PUT` з пустим тілом завжди передавати `{}`, інакше 400 Bad Request.
- Шляхи ендпоінтів згідно з Swagger (напр. `/restaurant`, не `/api/deliveries`).
- Purge Orders: спочатку DELETE, fallback на CANCEL при обмеженнях.
